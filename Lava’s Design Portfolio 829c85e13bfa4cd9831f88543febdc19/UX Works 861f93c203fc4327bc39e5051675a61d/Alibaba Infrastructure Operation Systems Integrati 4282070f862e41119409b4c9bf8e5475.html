Alibaba Infrastructure Operation Systems Integration

Time period2017
CompanyAlibaba Group_Infrustructure department
IndustryAlibaba CloudEfficiencySupply chain
RoleUX/UI designer
CategoryDesign SystemUX/UIWeb

Background:

There were more than 13 isolated systems relating to infrastructure Operation and management, run by different departures of Alibaba Group. With the development of database and backend technologies, these systems needed to be integrated together to be the sub products of a large product, so that all the relevant information could be accessed by different users within one portal.

1.Pain Points Analysis

The grand vision of this project is to redesign and recode all the systems, but obviously it cost much time and resource. The project team had to come out with the efficient solution, and made it work quickly.

So I arranged a group interview with the help with my design leader, and invited the leaders of each departments who was in charge of the isolated system, project manager, technical manager and users. All people wrote down their considerations and pain points on the walk. Each roles have their concerns for this project, thus for a designer, the goal is to provide a design solution with structure, organization and expandability.

2.Navigation System Design

Since all the system have different navigation hierarchy, the key point was to design a strong/flexible navigation system, to integrate so many systems.

Defined the guideline for each area.

3.Design core functions of each system

There are 13 sub-products need to be redesign, it’s impossible for one designer to redesign all of them with page by page. So I had to design the core functions with reusable layout for each sub-products.

4.First Launch

After 2 months, the first milestone came out with the first system being launched with the new design.

For the Sub-products, I help the owners of each system to clarify their core features which were used frequently by users, displaying together with the name of the Sub-products. Regarding of this, the rest of the Sub-products could just follow this structure to manage their information and features for integration.

5.Template Design

Design principles and templates for different scenarios/cases, so that non-designers could implemented their requirements on their own by following the instruction. The goal for that was to make greater difference to the team with the limitation of design resource.

6. Reflaction

The design process could be divided into 4 parts:

  • Navigation system design
  • Core functions design
  • Templates for the repeated scenario
  • Review the whole portal

Generally, we will always face the project that we can't complete on time or as planned. Given the un-normal and the large-scale project, if I follow the normal design process for example, design all these 13 systems by myself, it will cost a lot time and waste design resources.

As a designer, I think we could not only make the product look more pretty, but we could also think about the what kinds of the value we were able to bring to the whole process, and even make great differences to the project and the team.

Fortunately, the project was successful and the design made value on the project.
